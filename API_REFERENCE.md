# Commentum API Reference Specification

## Overview

The Commentum API exposes REST endpoints over HTTPS backed by Supabase Edge Functions. All endpoints exchange JSON payloads and require standard bearer token authentication for mutative actions.

**Base URL:** `https://<YOUR_PROJECT_REF>.supabase.co/functions/v1`

---

## Authentication & Headers

Protected endpoints require an `Authorization` header containing a valid JWT generated by the `/auth` service.

```http
Authorization: Bearer <JWT_TOKEN>
Content-Type: application/json
```

---

## Unified Data Model

All discussion items (comments and replies) share a unified relational schema within the `posts` database table. Hierarchical threading is maintained via recursive foreign keys and PostgreSQL trigger propagation.

### `Post` Object Schema

| Field | Type | Description |
| :--- | :--- | :--- |
| `id` | `string` (UUID) | Unique identifier for the post. |
| `client` | `string` \| `null` | Identifier of the client application that created the post. |
| `content` | `string` | The markdown or plain text content (max 500 characters). |
| `score` | `integer` | Net vote score (`upvotes - downvotes`). |
| `status` | `string` | Current moderation status (`active`, `hidden`, `removed`, `deleted`). |
| `created_at` | `string` (ISO 8601) | Timestamp of creation. |
| `updated_at` | `string` (ISO 8601) | Timestamp of last modification. |
| `parent_id` | `string` (UUID) \| `null` | Direct parent ID if this post is a reply. `null` for root comments. |
| `root_id` | `string` (UUID) | Ancestor root comment ID. For root comments, equals `id`. |
| `media_id` | `string` \| `null` | External media identifier (e.g., AniList ID). Inherited by replies. |
| `media_provider` | `string` \| `null` | Name of the media provider (`anilist`, `mal`, `simkl`). Inherited by replies. |
| `episode_number` | `integer` \| `null` | Episode number associated with the discussion. Inherited by replies. |
| `user` | `object` | Author profile containing `username` and `avatar_url`. |
| `replies` | `Post[]` \| `undefined` | Array of up to 5 preview replies (included only when listing root comments). |
| `has_more_replies` | `boolean` \| `undefined` | Indicates whether additional replies exist beyond the preview slice. |
| `replies_count` | `integer` \| `undefined` | Total exact reply count for the root comment thread. |
| `user_vote` | `integer` \| `null` | Current authenticated user's vote (`1`, `-1`, or `0` / `null`). |

## Endpoints Specification

### 1. Authentication Service (`/auth`)

#### `POST /auth`
Creates a user session using an external provider OAuth token.

- **Rate Limit:** 10 requests / minute per IP.
- **Request Body:**
  ```json
  {
    "provider": "anilist | mal | simkl",
    "access_token": "string"
  }
  ```
- **Response (200 OK):** Returns `{ "token": "jwt...", "user": { ... } }`.

#### `DELETE /auth`
Revokes the current active session.

- **Authentication:** Required.
- **Response (200 OK):** Returns `{ "message": "Logged out successfully" }`.

---

### 2. User Profile (`/me`)

#### `GET /me`
Retrieves the profile of the currently authenticated user.

- **Authentication:** Required.
- **Response (200 OK):** Returns `{ "user": { "id": "...", "username": "...", "avatar_url": "...", ... } }`.

---

### 3. Discussion Posts (`/posts`)

#### `GET /posts`
Retrieves paginated lists of comments or thread replies.

- **Query Parameters:**
  - `media_id` (*optional*): Fetch root comments for a media item. Must be paired with `episode_number` if filtering by episode.
  - `episode_number` (*optional*): Filter root comments by integer episode number. Pass `"0"` or `"none"` to filter for series-level comments lacking an episode assignment.
  - `root_id` (*optional*): Fetch all replies belonging to a root comment thread.
  - `parent_id` (*optional*): Fetch direct child replies for a specific parent post.
  - `limit` (*optional*, default: `20`, max: `100`): Number of items to return.
  - `cursor` (*optional*): ISO 8601 timestamp for cursor-based pagination (returns items created before the cursor).
- **Note:** Exactly one query strategy (`media_id`, `root_id`, or `parent_id`) must be provided.
- **Response (200 OK - Root Comments):**
  ```json
  {
    "comments": [ ... ],
    "comment_count": 42,
    "next_cursor": "2026-06-28T12:00:00.000Z"
  }
  ```
- **Response (200 OK - Replies):**
  ```json
  {
    "replies": [ ... ],
    "reply_count": 15,
    "next_cursor": null
  }
  ```

#### `POST /posts`
Creates a new root comment or reply.

- **Authentication:** Required.
- **Rate Limit:** 5 requests / minute per user (root comments), 10 requests / minute per user (replies).
- **Request Body (Root Comment):**
  ```json
  {
    "media_id": "101922",
    "media_provider": "anilist",
    "episode_number": 3,
    "content": "Discussion text...",
    "client": "shonenx"
  }
  ```
- **Request Body (Reply):**
  ```json
  {
    "parent_id": "uuid-of-parent",
    "content": "Reply text...",
    "client": "shonenx"
  }
  ```
- **Response (201 Created):** Returns `{ "post": { ... } }`.

#### `PATCH /posts`
Updates the text content of an active post owned by the authenticated user.

- **Authentication:** Required.
- **Rate Limit:** 10 requests / minute per user.
- **Request Body:** `{ "id": "uuid", "content": "Updated text..." }`
- **Response (200 OK):** Returns `{ "post": { ... } }`.

#### `DELETE /posts`
Soft-deletes a post owned by the authenticated user by setting its status to `deleted`.

- **Authentication:** Required.
- **Rate Limit:** 10 requests / minute per user.
- **Request Query or Body:** `?id=uuid` or `{ "id": "uuid" }`
- **Response (200 OK):** Returns `{ "post": { "id": "uuid", "status": "deleted", ... } }`.

---

### 4. Voting Service (`/votes`)

#### `POST /votes`
Casts or toggles an upvote or downvote on a post.

- **Authentication:** Required.
- **Rate Limit:** 30 requests / minute per user.
- **Request Body:** `{ "post_id": "uuid", "vote_type": 1 | -1 }`
- **Response (200 OK):** Returns `{ "post_id": "uuid", "score": 25 }`.

---

### 5. Moderation & Reporting (`/reports`, `/moderation-*`)

#### `POST /reports`
Reports a post for moderation review. Posts receiving 5+ reports are automatically hidden.

- **Authentication:** Required.
- **Rate Limit:** 5 requests / minute per user.
- **Request Body:** `{ "post_id": "uuid", "reason": "Reason details..." }`
- **Response (201 Created):** Returns `{ "message": "Report submitted" }`.

#### `GET /moderation-reports` (Moderator+)
Lists all unresolved user reports. Accepts `limit` and `offset` query parameters.

#### `POST /moderation-comment-status` (Moderator+)
Updates a post's status. Accepts `{ "comment_id": "uuid", "status": "active | hidden | removed" }`.

#### `POST /moderation-ban-user` (Admin Only)
Bans a user from participating in discussions. Accepts `{ "user_id": "uuid" }`.

## HTTP Status & Error Codes

| Status Code | Description |
| :--- | :--- |
| `200 OK` | The request succeeded. |
| `201 Created` | Resource created successfully. |
| `400 Bad Request` | Missing parameters, invalid JSON, or validation failure. |
| `401 Unauthorized` | Missing, expired, or revoked authentication token. |
| `403 Forbidden` | Insufficient privileges or account banned. |
| `404 Not Found` | The requested resource or parent post does not exist. |
| `429 Too Many Requests` | Rate limit exceeded. |
| `500 Internal Server Error` | Unhandled database or server exception. |
